# Requisitos funcionales

## RF01. Autenticación

El sistema debe permitir iniciar y cerrar sesión.

## RF02. Introducción de documentación funcional

El sistema debe permitir introducir documentación funcional en formato texto libre, DRF, DDS o combinación de ellos.

## RF03. Asociación de documentación a proyecto

El sistema debe permitir asociar la documentación introducida a un proyecto cuando dicha asociación no esté disponible inicialmente.

## RF04. Consulta de documentación registrada

El sistema debe permitir consultar la documentación registrada para un proyecto.

## RF05. Gestión de casos de uso locales

El sistema debe permitir extraer, listar, consultar, crear, actualizar y eliminar casos de uso locales.

## RF06. Gestión de requisitos funcionales locales

El sistema debe permitir extraer, listar, consultar, crear, actualizar y eliminar requisitos funcionales locales.

## RF07. Generación de escenarios Gherkin

El sistema debe permitir generar y listar escenarios Gherkin a partir de casos de uso y requisitos funcionales.

## RF08. Gestión de borradores

El sistema debe permitir crear, listar y consultar borradores de caso de prueba.

## RF09. Revisión y versionado de borradores

El sistema debe permitir añadir feedback a un borrador, regenerarlo, aceptarlo o rechazarlo.

## RF10. Generación de caso de prueba derivado de borrador

El sistema debe permitir generar un caso de prueba a partir de un borrador aceptado.

## RF11. Publicación en Kiwi TCMS

El sistema debe permitir publicar un caso de prueba en Kiwi TCMS.

## RF12. Consulta en Kiwi TCMS

El sistema debe permitir buscar y ver casos de prueba en Kiwi TCMS.

## RF13. Trazabilidad funcional

El sistema debe mantener trazabilidad entre proyecto, sesión, documentación, casos de uso, requisitos funcionales, escenarios Gherkin, borradores y casos de prueba.

## RF14. Gestión de sesiones de trabajo

El sistema debe permitir seleccionar sesiones existentes y crear nuevas sesiones para continuar o iniciar el trabajo en un contexto diferenciado.

## RF15. Almacenar/Guardar los resultados

El sistema debe permitir guardar los resultados generados o editados durante una sesión, incluyendo casos de uso, requisitos funcionales y borradores.

---

# Requisitos no funcionales
## RNF01. Consistencia

El sistema debe mantener coherencia entre los artefactos generados y gestionados durante todo el flujo funcional.

## RNF02. Trazabilidad

El sistema debe conservar referencias de origen y relaciones entre artefactos funcionales y de prueba.

## RNF03. Usabilidad

El sistema debe ofrecer operaciones claras, diferenciadas y comprensibles.

## RNF04. Integración

El sistema debe integrarse con Kiwi TCMS para registrar casos de prueba.

## RNF05. Mantenibilidad

El sistema debe organizar sus responsabilidades funcionales de manera modular.

## RNF06. Extensibilidad

El sistema debe permitir incorporar nuevas fuentes y nuevas operaciones sin alterar la estructura funcional general.

## RNF07. Identificación de contexto

El sistema debe preservar la relación entre cada artefacto y su proyecto de referencia.
